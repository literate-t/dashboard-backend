# 차량 관제 포트폴리오(서버)

## 소개

대규모 3D 교통 시뮬레이션의 백엔드 시스템으로, 수천 대의 차량 객체에 대한 경로 탐색, 실시간 이동 동기화, 클라이언트와의 양방향 통신을 담당합니다.

## 기술 스택

* **프레임워크:** Java 17+, Spring Boot 3.5.11, Spring Security 6.5.8
* **통신 방식:** WebSocket(STOMP), REST API
* **주요 알고리즘:** 최단 경로 탐색(다익스트라)

## 핵심 기술과 트러블슈팅

### 1. Payload Batching을 통한 네트워크 병목 최소화

* **이슈:** 수천 대의 차량이 100ms 단위로 각자의 위치 데이터를 개별적인 웹소켓 메시지로 브로드캐스트할 경우, 초당 수만 건의 I/O가 발생하여 서버 스레드 고갈 및
  클라이언트 브라우저 마비 현상이 발생할 수 있습니다.
* **해결:** 스케줄러(`@Scheduled`)를 통해 100ms 주기로, 활성화된 차량의 상태(진행률, 노선 정보 등)를 일괄 연산합니다. I/O가 전체 성능을 결정하기
  때문에, 모든 데이터를 단일 객체(`BatchedPositionPayload`)로 묶어 초당 10회만 전송하는 배칭(Batching) 기법을 적용해 통신 부하를 낮췄습니다.

### 2. 병렬(Parallel) 스트림을 활용한 경로 탐색 최적화

* **이슈:** 프론트엔드에서 수백 대의 차량 생성을 한 번에 요청할 때, 각 차량에 대한 최단 경로 탐색은 무거운 동작이므로 싱글 스레드에서 순차 순회를 하면 응답 지연이
  발생할 수 있습니다.
* **해결:** Java의 `parallelStream()`을 활용하여 여러 CPU 코어에 최단 경로 찾기 작업을 분배했습니다. 수많은 차량의 최단 경로를 거의 동시에 계산해
  대규모 트래픽 발생으로 인한 응답 지연을 최소화하고자 했습니다. 클라이언트에서 차량 500대씩을 추가 요청할 때, 싱글 스레드에서 수행했을 때보다 4 ~ 9배의 속도
  차이를 확인할 수 있었습니다.
